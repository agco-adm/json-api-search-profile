Multi-Resource Filtering and Multi-Level Aggregations.

The data model below represents a (simplified) view of a Dealer API domain and will be used throughout the document to further explain the concepts in this profile

Multi-Resource Filtering

The JSON API spec standardises filtering on a single resource. This profile adds the ability to apply filters on linked resources, which can be nested multiple levels deep.

Examples

Simple : Get all dealers located in the state of NY

/dealers?state_province.code=US-NY

Linked Resource : Get all dealers with a zip code of 10005 and an 'MFP' contract code

/dealers?zip=10005&current_contracts.code=MFP

Multiple Linked Resources : Get all dealers located in the state of NY and have a contract to sell MF equipment

/dealers?state_province.code=US-NY&current_contracts.brand.code=MF
